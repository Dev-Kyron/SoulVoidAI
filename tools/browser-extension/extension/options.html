VoidSoul Browser Extension

Local-only bridge. Every request goes through the desktop app on this machine. Nothing crosses the network unless the underlying provider you picked in the desktop app does that itself.

One-time setup

  1. Open the VoidSoul desktop app.
  2. Go to Settings → Tools → Browser Extension and flip Enable bridge on.
  3. Copy your extension id from chrome://extensions (developer mode on) and paste it into the Settings panel's "Install" prompt.
  4. Restart your browser. Done.

The Settings panel's Copy install command button writes the right shell command for your OS — Mac/Linux register a JSON manifest, Windows also writes a registry key under HKCU\Software\Google\Chrome\NativeMessagingHosts.

Daily use

Hotkey: Alt+Shift+J. Highlight some text first, hit the hotkey, the overlay opens with that selection as context. Type a question, press Enter, watch the reply stream in. Esc closes the overlay.

You can also right-click any selection and pick Ask VoidSoul about “…” — same surface, same context.

Privacy

The extension talks to the desktop app through Chrome's nativeMessaging API over a local stdio pipe — no network sockets, no fetch, no remote server. The host manifest only allows chrome-extension://<your-id>/ as a sender, so other extensions can't piggyback on the bridge.

The desktop app's local IPC socket sits at a per-OS path (~/Library/Application Support/VoidSoul AI Companion/voidsoul-extension.sock on macOS, an equivalent Windows named pipe), with file permissions scoped to your user account.
