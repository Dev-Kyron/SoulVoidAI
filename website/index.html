What it does

Everything a local-first AI should do.

One install, every capability ready to flip on. No add-on store. No per-seat charge.

Voice in, voice out

Piper TTS for replies, Whisper or Porcupine for the wake word, full-duplex conversation mode for back-and-forth. Pick a different voice per persona. Tone markers in the prompt drive delivery.

Sees your screen

Continuous awareness of the focused window plus local OCR on every change — the assistant knows you're editing a specific file or reading a specific tab without you having to copy-paste.

Clicks for you

Describe a click in plain English. The pipeline tries taught shortcuts, then Sonnet computer-use, then UIA, then vision. A 3-second cancellable preview ring fires before the click so wrong locates fail safe.

Runs Python

Each chat thread gets its own persistent Jupyter-style kernel. State, imports and plots survive across tool calls within the same conversation. Workspace dirs are real folders you can open.

Speaks unprompted

Cron-fired briefings and condition-driven watch tasks (idle duration, sentiment shifts, task completion) let Soul speak when something useful happens — not just when you ask.

Talks to your stuff

Home Assistant native integration, MCP servers from curated and community marketplaces, scheduled tasks, browser-extension bridge for active-tab tools — all in one panel.

Spotlight

click_on_screen — five strategies, one tool call.

Describe a click. The pipeline picks the cheapest strategy that works. Teach it once and the next identical click fires in milliseconds with no model spend.

1 · Taught

Teach a click once

Point your cursor at any button, press F8, type what to call it ("send in slack"). Future identical descriptions short-circuit to a direct UIA click. Zero model tokens, instant.

2 · Sonnet computer-use

Grounded clicks from Claude

When you're on a capable Anthropic Sonnet, the pipeline auto-routes through native computer-use — wins on browser content and icon-only buttons where free-form vision mis-clicks by 50-200 pixels.

3 · UIA match

Accessibility-tree first

Reads the Windows UIA tree, matches the user's description against accessibility names. The fast path for any native app with proper labels — Discord, Slack, Office, VS Code, File Explorer.

4 · UIA candidate pick

Vision picks from a list

When UIA enumerated candidates but none matched, the vision model picks an id from the list with the screenshot as context. Coords come from UIA's bbox so accuracy is pixel-exact when the pick is right.

5 · Vision locate

Free-form fallback

Two-pass refinement — first pass picks a region, second pass zooms in for the precise point. Authoritative refinement (refused refinement = no click) prevents the confidently-wrong failure mode.

Safety

3-second cancellable preview

Every click is preceded by a translucent cyan ring at the target. Press Esc to abort. Wrong locates fail safe; nothing fires until you let the countdown finish.

Under the hood

Built for power users who don't want a SaaS.

Provider freedom — bring any LLM, mix and match.

OpenAI, Anthropic, Gemini, Groq, xAI, OpenRouter, Mistral, DeepSeek, Ollama, LM Studio, llama.cpp — all first-class. API keys are encrypted with your OS keychain (DPAPI on Windows, Keychain on macOS, libsecret on Linux) and never leave this machine. Auto-fallback handles 429s and 5xxs by retrying on a different provider. Auto-router can switch providers mid-prompt for speed or cost. Or turn both off and every send goes verbatim to your active pick. 

provider: anthropic
model:    claude-sonnet-4-5
fallback: openrouter / openai
local:    ollama (✓ detected)

Memory in three layers.

Explicit facts you teach the AI; passive biographical extracted in the background from your conversations; emotional context from a daily sentiment classifier that biases tone. Edit, scope to specific modes, or wipe entirely. The summariser knobs control when older messages roll into a "story so far" recap — raise the trigger for coding sessions where recent edits matter, lower it for creative writing where arc continuity wins.

RAG over your files, plus a persistent Python kernel.

Point at folders; files get chunked, embedded with your picked engine (OpenAI, Ollama, or fully-local Transformers.js), and stored in SQLite. The vector-store browser shows you which chunks landed and which ones got retrieved for the most recent reply — useful when an answer feels off. Each chat thread also gets its own long-running Python kernel: variables, imports, plots all survive across tool calls. 

# In thread A
import pandas as pd
df = pd.read_csv('sales.csv')

# Next turn, same thread:
df.groupby('region').sum()
# → still has the DataFrame

Proactive when it should be.

Cron-fired tasks for daily briefings and scheduled research. Watch tasks fire on conditions: you went idle for N minutes, your sentiment shifted, a long-running tool finished. Soul speaks unprompted with a short, skippable line — Do Not Disturb mutes voice and dims the orb during your quiet hours. The same scheduler bridges into deep-research mode: kick off a "study this market" task at 8am and a synthesised brief lands in a new chat thread when it's done.

Common questions

Quick answers.

Is this a subscription?

No. Core is free + open source. The signed installer is a one-time $29. You can switch back to a self-built binary any time without losing data.

Do my conversations leave my machine?

Only when you use a cloud LLM provider — those calls go directly from your machine to that provider's API. VoidSoul has no backend. We can't see your prompts; we don't relay them.

Will it click in apps I don't trust?

Every click_on_screen action shows a 3-second cancellable preview ring. The inputAccess permission can be revoked at any time, after which the AI can't drive your mouse at all.

What if I'm offline?

Local providers (Ollama, LM Studio, llama.cpp) work entirely offline. Voice (Piper / Whisper), OCR (tesseract.js), and RAG embeddings (Transformers.js mini-LM) can all run with no network too.

What's the difference between Soul and Void?

Personas. Pick either one as the default voice, swap per-thread, or build your own custom persona bundle. Same engine underneath.
